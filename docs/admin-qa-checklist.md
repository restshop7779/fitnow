# FitNow 관리자 QA 체크리스트

이 문서는 총관리자 화면의 테스트 도구, 정산 플로우 점검, 리포트, 테스트 데이터 정리 기능을 배포 전 수동으로 확인하기 위한 체크리스트입니다.

## 사전 준비

- 로컬 앱 실행: `http://127.0.0.1:5173/index.react.html`
- 권장 확인 브라우저: 데스크톱 1280px 이상, 모바일 390px 전후
- 테스트 전 가능하면 `테스트 데이터 정리`를 먼저 실행해 운영 모드에서 시작합니다.
- 문서 기준 빌드 명령: `.\node_modules\.bin\vite.cmd build`

## 1. 총관리자 기본 상태

- [ ] 총관리자 화면에 진입했을 때 운영/테스트 모드 배너가 표시된다.
- [ ] 테스트 데이터가 없으면 배너가 `OPERATING MODE`로 표시된다.
- [ ] 테스트 데이터가 있으면 배너가 `TEST MODE`로 표시되고 `정리` 버튼이 보인다.
- [ ] `기사별 업무판`은 기본 접힌 상태로 표시된다.
- [ ] `기사 닉네임 관리`는 기본 접힌 상태로 표시된다.
- [ ] 운영 홈의 할 일 카드와 포커스 배너가 겹치지 않는다.

## 2. 관리자 테스트 도구

- [ ] `관리자 테스트 도구` 섹션은 기본 접힌 상태다.
- [ ] 섹션을 펼치면 최근 점검 시간, 최근 정리 시간이 표시된다.
- [ ] 보존 기간 선택값 `1시간`, `24시간`, `7일` 버튼이 표시된다.
- [ ] 기본 보존 기간은 `24시간`이다.
- [ ] 보존 기간을 변경하면 테스트 모드 배너의 자동 정리 문구가 같은 값으로 갱신된다.
- [ ] 테스트 도구 버튼들이 화면 하단 밖으로 밀려나지 않는다.

## 3. 정산 플로우 점검

- [ ] `정산 플로우 점검`을 클릭하면 테스트 주문 `FN-SET001`이 생성된다.
- [ ] 동기화 메시지에 `정산 플로우 자동 점검 완료 - FN-SET001 지급완료`가 표시된다.
- [ ] 정산 결과 요약에 지급대기에서 지급완료로 전환된 내역이 표시된다.
- [ ] 지급완료 탭이 활성화된다.
- [ ] 지급완료 목록에 `FN-SET001` 행이 강조 표시된다.
- [ ] `정산 플로우 점검 로그`에 최근 로그 1건 이상이 표시된다.
- [ ] 최근 점검 시간이 현재 실행 시각으로 갱신된다.
- [ ] 테스트 데이터가 생긴 뒤 배너가 `TEST MODE`로 전환된다.

## 4. 정산 점검 리포트

- [ ] `최근 리포트` 버튼을 클릭하면 리포트 모달이 열린다.
- [ ] 로그별 `상세보기` 버튼을 클릭해도 같은 주문의 리포트가 열린다.
- [ ] 모달 제목이 `정산 점검 리포트`로 표시된다.
- [ ] 리포트에 다음 4단계가 모두 표시된다.
  - 테스트 주문 생성
  - 정산 확정
  - 지급 완료
  - 화면 반영
- [ ] `텍스트 복사` 버튼이 표시되고 클릭 가능하다.
- [ ] `CSV 다운로드` 버튼이 표시되고 클릭 가능하다.
- [ ] 모달 닫기 후 기존 정산 화면 상태가 유지된다.

## 5. 테스트 데이터 정리

- [ ] `테스트 데이터 정리`를 클릭하면 진단 주문과 정산 점검 로그가 제거된다.
- [ ] `FN-SET001`이 정산 목록에서 사라진다.
- [ ] 정산 점검 로그가 빈 상태 메시지로 돌아간다.
- [ ] 최근 정리 시간이 현재 실행 시각으로 갱신된다.
- [ ] 배너가 `OPERATING MODE`로 전환된다.
- [ ] 운영 홈의 테스트 데이터 안내가 사라진다.

## 6. 배송 플로우 점검

- [ ] 운영 홈의 배송 플로우 자동 점검 버튼을 실행할 수 있다.
- [ ] 테스트 주문 `FN-TEST-FLOW` 계열 데이터가 생성된다.
- [ ] 점검 결과가 완료 상태로 표시되거나, 확인 필요 상태가 명확히 안내된다.
- [ ] 배송 점검 후에도 테스트 데이터 정리로 관련 데이터가 제거된다.

## 7. 회귀 확인

- [ ] 일반 주문 목록, 주문 상세 모달, 기사 배정 동작이 기존처럼 동작한다.
- [ ] 정산서 미리보기, CSV 생성, 마감 버튼이 테스트 도구 섹션 밖의 운영 액션 영역에 남아 있다.
- [ ] 테스트 도구를 접어도 운영 액션 버튼은 사용할 수 있다.
- [ ] 모바일 폭에서 배너, 테스트 도구, 버튼 텍스트가 서로 겹치지 않는다.
- [ ] 새로고침 후에도 보존 기간 설정과 최근 점검/정리 시간이 유지된다.

## 알려진 확인 제한

- Codex 인앱 브라우저에서는 실제 파일 다운로드 완료 이벤트를 검증하기 어렵다. `CSV 다운로드`는 버튼 렌더링과 클릭 가능 상태를 확인하고, 실제 다운로드는 일반 브라우저에서 한 번 더 확인한다.
- 문서만 수정한 경우 앱 빌드는 필수는 아니지만, 배포 전에는 `.\node_modules\.bin\vite.cmd build`를 실행한다.
