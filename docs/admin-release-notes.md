# FitNow Admin Release Notes

작성일: 2026-06-16

## 범위

이번 릴리즈는 총관리자 운영 화면의 정산 점검, 테스트 데이터 관리, QA 체크리스트, 배포 준비 상태 확인 흐름을 하나의 운영 점검 루프로 묶는 작업입니다.

## 주요 변경

### 관리자 테스트 도구

- `관리자 테스트 도구` 섹션을 정산 운영 액션과 분리했습니다.
- 섹션은 기본 접힘 상태로 유지됩니다.
- 테스트 도구 안에서 다음 기능을 실행할 수 있습니다.
  - 정산 플로우 점검
  - QA 체크리스트
  - 엑셀 테스트 6건 생성
  - 테스트 데이터 정리
- 최근 점검, 최근 정리, QA 체크 진행률을 테스트 도구 상단에 표시합니다.
- 테스트 데이터 보관 기간을 `1시간`, `24시간`, `7일` 중 선택할 수 있습니다.

### 정산 플로우 점검

- `정산 플로우 점검` 실행 시 테스트 주문 `FN-SET001`을 생성합니다.
- 확정대기에서 정산 확정, 지급완료까지 자동으로 처리합니다.
- 지급완료 탭으로 전환하고 결과 요약과 로그를 남깁니다.
- `최근 리포트`와 로그별 `상세보기`에서 정산 점검 리포트를 확인할 수 있습니다.
- 정산 점검 리포트는 텍스트 복사와 CSV 다운로드 버튼을 제공합니다.

### 테스트 데이터 정리

- 진단 주문 기준:
  - `FN-TEST-*`
  - `FN-SET*`
- 정리 대상:
  - 로컬 주문 이력
  - 주문 상태 저장값
  - 정산 상태 저장값
  - 정산 플로우 점검 로그
  - 테스트 관련 마지막 정산 결과
- 테스트 데이터가 남아 있으면 총관리자 배너가 `TEST MODE`로 표시됩니다.
- 정리 후 테스트 데이터가 없으면 `OPERATING MODE`로 전환됩니다.
- 새로고침 후 DB에서 진단 주문이 다시 노출되던 문제를 수정했습니다.
  - 관리자 주문 로딩에서 진단 주문을 제외합니다.
  - 업체 주문 로딩에서도 진단 주문을 제외합니다.

### 관리자 QA 체크리스트

- `QA 체크리스트` 버튼을 관리자 테스트 도구에 추가했습니다.
- 체크리스트는 기존 관리자 상세 모달을 재사용합니다.
- 6개 섹션, 24개 항목으로 구성됩니다.
- 체크 상태는 `localStorage`에 저장됩니다.
- 모달을 닫았다 다시 열어도 체크 상태가 유지됩니다.
- `체크 초기화`로 저장된 QA 상태를 초기화할 수 있습니다.
- QA 리포트는 텍스트 복사와 CSV 다운로드를 지원합니다.

### 배포 준비 상태 패널

- 총관리자 상단에 `배포 준비 상태` 패널을 추가했습니다.
- 다음 4개 항목을 요약합니다.
  - QA
  - 테스트 데이터
  - 최근 점검
  - 최근 정리
- 준비 상태는 `0/4`부터 `4/4`까지 표시됩니다.
- 모든 항목이 충족되면 `배포 전 필수 점검 완료 4/4`로 표시됩니다.
- 미완료 항목이 있으면 `배포 전 확인 항목이 남아 있습니다`로 표시됩니다.

## 배포 전 권장 순서

1. 총관리자 화면 진입
2. 상단 `QA 체크리스트`, `정산서 미리보기` 바로가기 위치 확인
3. `배송 대시보드 > 지금배송 정산 요약 > 관리자 테스트 도구` 펼치기
4. `정산 플로우 점검` 실행
5. 상단 또는 테스트 도구의 `QA 체크리스트`로 24개 항목 확인
6. `리포트 복사` 또는 `CSV 다운로드`로 QA 기록 저장
7. `테스트 데이터 정리` 실행
8. 상단 `배포 준비 상태`가 `4/4`인지 확인
9. 새로고침 후에도 `OPERATING MODE`와 `4/4`가 유지되는지 확인
10. `.\node_modules\.bin\vite.cmd build` 실행
